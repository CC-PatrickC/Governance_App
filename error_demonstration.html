Issue Summary

Database Migration Issue

Problem: The "projects_triagechange" table was missing from the Azure PostgreSQL database, causing the edit form to fail when trying to load project change history.

Root Cause: Database migrations were not run on the Azure deployment, so the table structure was out of sync with the application code.

Solution: Ran python manage.py migrate in Azure SSH console to apply missing database migrations.

Status: Issue resolved successfully!

Technical Details

Missing Migration

Migration 0034_add_triage_change_model.py was not applied to Azure database.

This migration creates the projects_triagechange table needed for tracking project changes.

Enhanced error handling was added to provide detailed error messages instead of generic "Error loading edit form".
